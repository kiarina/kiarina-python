---
title: Rules for package root README.md
description: Guidelines for creating README.md files at the root of package.
---

When creating `packages/{package_name}/README.md`,
please summarize the following content as simply as possible to minimize the amount of text.

- The purpose of the package
- Main usage of the package
- Explanation of how to use public objects (classes, functions, types, etc.)
- Configuration explanation (pydantic_settings, pydantic_settings_manager)

For example, use the following structure:

- Purpose
- Installation
- Quick Start
- API Reference
- Configuration: Configuration examples should be in YAML format only
- Testing
- Dependencies
- License
- Related Projects
